# 设计：Memory-Master - OpenClaw 超级记忆增强 Skill

**生成时间**：2026-04-06  
**模式**：Builder（开源项目）  
**作者**：小鬼 👻 × Jake

---

## 问题陈述

OpenClaw 的记忆力极差——每次 `/new` 或重启后，AI 像失忆一样完全不记得上次聊了什么。现有系统（MEMORY.md + daily notes）是被动的，依赖 agent 主动写入，但 agent 经常忘记。

**核心痛点**：跨会话记忆不连续，用户每次重启都得重新解释上下文。

---

## 这个想法酷在哪

1. **像人一样记忆** - 不是简单存储，而是会过滤、压缩、关联、整理
2. **完全自动化** - 用户无需手动操作，AI 自动捕捉重要信息
3. **记忆图谱** - 可视化记忆关联，像大脑神经元一样连接
4. **定期整理** - 像睡觉时整理记忆，后台自动提炼/合并/删除

---

## 约束条件

- **Token 成本** - 不能无脑全记，需要智能过滤和压缩
- **启动速度** - 重启后加载记忆不能太慢（<5 秒）
- **现有系统兼容** - 不能破坏现有的 MEMORY.md + daily notes 机制
- **Skill 体积** - 保持轻量，避免过度复杂

   - 混合方式？（摘要 + 关键引用）

3. **记忆图谱存储** - 用什么格式存储关联关系？

   - SQLite？（查询快，支持复杂关系）

4. **重启检测机制** - 如何知道是"新会话"需要加载记忆？
   - 检测会话 ID 变化？
   - 检测 `/new` / `/reset` 命令？

### 重要性评分算法（草案）

```javascript
// 基于规则和语义的混合评分
function scoreImportance(message) {
  let score = 0;
  
  // 规则 1: 包含关键词（项目、决策、待办、用户信息）
  const keywords = ['项目', '决定', '待办', '记住', '重要', '偏好', '习惯'];
  keywords.forEach(kw => {
    if (message.includes(kw)) score += 10;
  });
  
  // 规则 2: 用户明确说"记住这个"
  if (message.match(/记住 | 记下来 | 别忘了/)) score += 30;
  
  // 规则 3: 包含具体数据（日期、数字、URL）
  if (message.match(/\d{4}-\d{2}-\d{2}/)) score += 5; // 日期
  if (message.match(/https?:\/\//)) score += 5; // URL
  
  // 规则 4: 是对话的总结/决策
  if (message.match(/总之 | 所以 | 决定 | 结论/)) score += 15;
  
  return score;
}

// 阈值：score > 50 写入记忆
```
